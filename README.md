## рҹҡҖ What's New in v7 вҖ” Everything Changed

> v6 was a working prototype. v7 is a deployable platform.

### рҹ”ҙ Critical Bugs Fixed (would have crashed v6)

| Bug | Impact | Fix |
|-----|--------|-----|
| `OtpToken`, `Rating`, `CoReporter` were package-private | **Compile failure** вҖ” app wouldn't even build | Made all 3 `public class` |
| `updateStatus()` called `auth.me()` in WhatsApp webhook | **NullPointerException** вҖ” webhook has no JWT, crashes silently | Actor now passed explicitly as parameter |
| `ddl-auto=validate` on empty Render database | **App refuses to start** on fresh deploy | Changed to `none` + auto schema init |
| `schema.sql` not on Java classpath | Spring couldn't find schema to run | Copied to `src/main/resources/` |
| Render gives `postgres://` URL, Spring needs `jdbc:postgresql://` | **Database connection fails** on cloud | `render.yaml` startCommand converts URL via `sed` |
| Duplicate email in seed data (Naman Agrawal Г— 3) | **PostgreSQL UNIQUE constraint crash** on schema import | Faculty/admin use separate institutional emails |
| Hardcoded `localhost:8080` in both HTML files | Frontend talks to offline PC when deployed | `config.js` вҖ” change one line, both pages update |
| No `IF NOT EXISTS` on any CREATE TABLE | Schema fails on re-run | Added to all 10 tables + 10 indexes |
| No `.nojekyll` file | GitHub Pages Jekyll processor silently breaks paths | Added вҖ” tells GitHub to serve files as-is |
| No `index.html` at repo root | GitHub Pages 404 on base URL | Added redirect index at root |
| Missing `spring-boot-starter-actuator` | `/actuator/health` endpoint 404 вҖ” Render health check fails | Added to `pom.xml` + config |

### рҹҹЎ Logic & Ambiguity Bugs Fixed

| Bug | Impact | Fix |
|-----|--------|-----|
| WhatsApp webhook picked "most recent" complaint | Wrong complaint updated when staff has multiple open | Now parses `#REF` from reply e.g. `DONE #EL-1001` |
| Phone number format mismatch | `findByWhatsappNumber()` always returned empty | Normalized to 12-digit, with 10-digit fallback |
| Webhook completely unauthenticated | Anyone could POST YES/DONE and change status | Token validation header added |
| `assign()` never sent WhatsApp | Staff never notified when admin assigns complaint | WhatsApp sent to assigned staff member's number |
| `@PreAuthorize` used lowercase roles | Spring Security stores `ROLE_ADMIN` вҖ” lowercase always returned 403 | All annotations updated to uppercase |

### рҹҹў New Features Added

| Feature | What it does |
|---------|-------------|
| **Repair Proof Upload** | Staff uploads photo after fixing вҖ” student sees proof before confirming closure |

## рҹ§  Smart Features Explained

### 1. Intelligent Ticket Routing
```
User types: "Projector not working in LT 101"
              вҶ“
    Backend classifyCategory() scans keywords
              вҶ“
    Matches: "projector" вҶ’ category = technical
              вҶ“
    AUTO_ROUTE contains "technical" вҶ’ WhatsApp sent to IT staff directly
              вҶ“
    Staff replies: "YES #IT-1001" вҶ’ status = in_progress
    Staff replies: "DONE #IT-1001" вҶ’ status = resolved вҶ’ OTP emailed to student
```

### 2. Priority Detection
```
"sparking wire"  вҶ’  critical  (safety risk keywords)
"water leakage"  вҶ’  high      (damage risk keywords)  
"chair broken"   вҶ’  medium    (standard)
Faculty files any complaint вҶ’ auto-elevated to HIGH minimum
5+ students report same issue вҶ’ auto-escalated + admin alerted
3 reminders sent вҶ’ auto-escalated
```

### 3. Smart WhatsApp Routing (NEW in v7)
```
Electrical в”Җв”ҖвҶ’ WhatsApp to electrician directly   (auto-route)
Plumbing   в”Җв”ҖвҶ’ WhatsApp to plumber directly        (auto-route)

Furniture  в”Җв”ҖвҶ’ Admin notified вҶ’ Admin assigns manually
Infrastructure вҶ’ Admin notified вҶ’ Admin assigns manually
Other      в”Җв”ҖвҶ’ Admin notified вҶ’ Admin assigns manually
```

## рҹ”§ Troubleshooting

| Symptom | Cause | Fix |
|---------|-------|-----|
| OTP not received | No Gmail config | Check console вҖ” OTP printed there in dev mode |
| Enrollment not found | Wrong ID format | IDs are uppercase e.g. `0801CA251084` |
| Frontend shows Network Error | Backend sleeping (Render free tier) | Wait 30 sec вҖ” or open health URL first |
| CORS error in browser | `FRONTEND_URL` env var mismatch | Must exactly match your GitHub Pages URL |
| Backend won't start on Render | Wrong DB URL format | Render gives `postgres://` вҖ” our `render.yaml` converts it automatically |
| 404 on GitHub Pages | Files uploaded in wrong folder structure | Use `campusfix_GITHUB_UPLOAD.zip` вҖ” files are at root level |
| Schema errors on first run | Tables already exist | Safe вҖ” all CREATE statements have `IF NOT EXISTS` |
| WhatsApp YES/DONE not working | Staff phone not in DB | Add whatsappNumber in Admin вҶ’ Users tab |
